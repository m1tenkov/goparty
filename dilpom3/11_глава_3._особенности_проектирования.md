# ГЛАВА 3. ОСОБЕННОСТИ ПРОЕКТИРОВАНИЯ

## 3.1. Обоснование выбора инструментария

При написании квалификационной работы были использованы следующие инструменты и программные средства:

Cloud.ru Evolution: использовалась в качестве облачной платформы для размещения серверной части проекта. На ней было организовано окружение для запуска ВК-бота и подключения к удаленной базе данных. Платформа была выбрана как средство развертывания удаленной инфраструктуры, обеспечивающей доступность приложения вне локальной среды разработки.

Python (версия 3.13.6): язык программирования, использованный для разработки серверной логики ВК-бота, обработки событий платформы «ВКонтакте», реализации пользовательских сценариев, взаимодействия с базой данных и логирования работы системы. Выбор Python обусловлен его распространенностью, большим количеством библиотек и удобством разработки серверных приложений [5; 6].

VK API: программный интерфейс платформы «ВКонтакте», использованный для интеграции бота с социальной сетью. С его помощью реализована обработка входящих сообщений [18], отправка ответов пользователям, работа с кнопками, получение открытых данных профиля и организация диалогового взаимодействия [17; 18].

MySQL: реляционная система управления базами данных, использованная для хранения структурированной информации приложения. В базе данных размещаются анкеты пользователей, игровые предпочтения, фотографии, история взаимодействий, лайки, мэтчи, данные пользовательских сессий и служебные признаки модерации [2-4; 15].

DBeaver: инструмент администрирования баз данных, использованный для подключения к удаленной MySQL-базе данных, просмотра таблиц, проверки структуры хранимых данных, выполнения
SQL-запросов и контроля содержимого базы на этапах разработки и тестирования [2].

XAMPP Control Panel и phpMyAdmin: использовались в локальной среде разработки для работы с MySQL, первичной настройки базы данных, выполнения тестовых запросов, визуального контроля таблиц и проверки корректности хранения информации.

Visual Studio Code: интегрированная среда разработки, использованная как основной инструмент для написания, редактирования и сопровождения программного кода проекта. Среда была выбрана благодаря удобному интерфейсу, поддержке Python и большому количеству расширений для разработки [1].

ППП MS Office:

Microsoft Word – для написания текста квалификационной работы.

Microsoft PowerPoint – для подготовки презентации к защите.

Git и GitHub: использовались для контроля версий проекта, хранения исходного кода и переноса актуальной версии приложения из локальной среды разработки на удаленный сервер. Применение Git и GitHub позволило фиксировать изменения в коде, упростить сопровождение проекта и обеспечить удобное развертывание серверной версии бота [21].

Сайт draw.io: веб-инструмент для построения диаграмм и схем, использованный для создания диаграмм, отражающих архитектуру и бизнес-процессы разрабатываемой системы. Инструмент был выбран благодаря бесплатному доступу, удобству работы и поддержке экспорта диаграмм в различные форматы.
