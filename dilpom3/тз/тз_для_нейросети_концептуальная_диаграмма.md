# ТЗ для нейросети на генерацию концептуальной диаграммы БД

## Что нужно получить

Нужно сгенерировать изображение концептуальной ER-диаграммы базы данных ВК-бота "ГоПати". Диаграмма будет вставлена в диплом в раздел 3.3 "Проектирование структуры БД" перед подразделом "Нормализация базы данных".

Изображение должно быть аккуратным, читаемым, в учебном стиле, без лишних декоративных элементов. Фон белый, линии тонкие, блоки сущностей прямоугольные.

## Готовый запрос для нейросети

Сгенерируй концептуальную ER-диаграмму базы данных информационной системы ВК-бота "ГоПати".

Диаграмма должна быть на русском языке. Стиль - строгий академический, для вставки в дипломную работу. Фон белый. Используй прямоугольники для сущностей и линии для связей. Укажи кратности связей: 1:1, 1:M, M:N.

На диаграмме должны быть следующие сущности:

1. Пользователь
2. Профиль пользователя
3. Игра
4. Игровые предпочтения
5. Фотография пользователя
6. Взаимодействие
7. Совпадение
8. Отложенный лайк
9. Пользовательская сессия

Покажи следующие связи:

1. Пользователь - Профиль пользователя: связь 1:1.
2. Пользователь - Фотография пользователя: связь 1:M.
3. Пользователь - Игровые предпочтения: связь 1:1.
4. Пользователь - Игра: связь M:N через сущность "Игры пользователя" или через связь с игровыми предпочтениями.
5. Пользователь - Взаимодействие: связь 1:M, так как пользователь может выполнять много действий по отношению к другим пользователям.
6. Взаимодействие связывает двух пользователей: отправителя и получателя.
7. Пользователь - Совпадение: связь 1:M, так как пользователь может иметь несколько взаимных совпадений.
8. Совпадение связывает двух пользователей.
9. Пользователь - Отложенный лайк: связь 1:M, так как пользователь может отправлять и получать отложенные лайки.
10. Отложенный лайк связывает отправителя и получателя.
11. Пользователь - Пользовательская сессия: связь 1:1.

Внутри сущностей можно указать только основные атрибуты, без типов данных:

Пользователь:
- id
- vk_user_id
- created_at

Профиль пользователя:
- user_id
- имя
- возраст
- город
- пол
- описание

Игра:
- id
- код игры
- название игры

Игровые предпочтения:
- user_id
- цель поиска
- сортировка
- возрастной диапазон
- предпочитаемый микрофон
- требуемая игра

Фотография пользователя:
- id
- user_id
- ссылка на фото
- порядок отображения

Взаимодействие:
- id
- отправитель
- получатель
- действие
- дата создания

Совпадение:
- id
- первый пользователь
- второй пользователь
- дата создания

Отложенный лайк:
- id
- отправитель
- получатель
- сообщение
- дата создания

Пользовательская сессия:
- user_id
- данные сессии
- дата обновления

Диаграмма должна показывать логическую структуру предметной области, а не физическую модель базы данных. Не нужно указывать типы данных, индексы, ограничения, SQL-команды и технические детали реализации. Главное - показать сущности и связи между ними.

Сделай изображение горизонтальным, чтобы его было удобно вставить в диплом на страницу альбомной ориентации или уменьшить до ширины страницы. Все надписи должны быть хорошо читаемыми.

## Дополнительная инструкция для нейросети

Если нейросеть плохо справляется с большим количеством текста внутри блоков, можно оставить только названия сущностей и кратности связей. В таком случае приоритет важнее у связей между сущностями, чем у списка атрибутов.

## Куда вставить изображение в диплом

Изображение нужно вставить в файл:

13_3.3._проектирование_структуры_бд.md

Вставлять после абзаца:

"Архитектура базы данных проекта реализована на основе реляционной модели. В структуре БД выделены основные сущности, отражающие логику работы приложения: пользователи, профили, игры, связи пользователей с играми, фотографии, взаимодействия между пользователями, взаимные совпадения, отложенные лайки и пользовательские сессии. Такое разделение позволяет избежать дублирования информации, упростить сопровождение системы и повысить прозрачность логики хранения данных."

Сразу после этого абзаца нужно вставить изображение концептуальной диаграммы.

## Подпись под изображением

Под изображением нужно указать:

Рисунок 7 - Концептуальная модель базы данных ВК-бота "ГоПати"

## Текст после изображения

После подписи можно оставить следующий переходный текст:

На концептуальной модели представлены основные сущности предметной области и связи между ними. Центральной сущностью является пользователь, с которым связаны профиль, фотографии, игровые предпочтения, пользовательская сессия, взаимодействия, отложенные лайки и совпадения. Такая структура отражает основные сценарии работы ВК-бота и служит основой для дальнейшей нормализации базы данных.

## Важное замечание по нумерации рисунков

Так как новая концептуальная диаграмма будет вставлена перед уже существующей физической моделью, текущую подпись:

Рисунок 7 - Физическая модель базы данных ВК-бота "ГоПати"

нужно заменить на:

Рисунок 8 - Физическая модель базы данных ВК-бота "ГоПати"
