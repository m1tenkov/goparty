# Техническое задание на создание концептуальной диаграммы базы данных

## Назначение диаграммы

Необходимо подготовить концептуальную диаграмму базы данных информационной системы ВК-бота "ГоПати". Диаграмма должна быть вставлена в раздел 3.3 "Проектирование структуры БД" перед подразделом "Нормализация базы данных".

Цель диаграммы - показать основные сущности предметной области и связи между ними на логическом уровне до описания нормализации и физической модели базы данных.

## Объект моделирования

Моделируется база данных ВК-бота "ГоПати", предназначенная для хранения пользователей, анкет, игровых интересов, фотографий, взаимодействий, совпадений, отложенных лайков и пользовательских сессий.

## Требования к диаграмме

На концептуальной диаграмме необходимо отразить следующие сущности:

1. Пользователь.
2. Профиль пользователя.
3. Игра.
4. Игровые предпочтения пользователя.
5. Фотография пользователя.
6. Взаимодействие между пользователями.
7. Совпадение пользователей.
8. Отложенный лайк.
9. Пользовательская сессия.

## Основные связи между сущностями

На диаграмме необходимо показать следующие связи:

1. Один пользователь имеет один профиль.
2. Один пользователь может иметь несколько фотографий.
3. Один пользователь может выбрать несколько игр.
4. Одна игра может быть выбрана несколькими пользователями.
5. Один пользователь может выполнять множество взаимодействий с другими пользователями.
6. Взаимное положительное взаимодействие пользователей образует совпадение.
7. Один пользователь может отправлять и получать отложенные лайки.
8. Один пользователь может иметь связанную пользовательскую сессию.
9. Игровые предпочтения пользователя уточняют параметры поиска и интересы пользователя.

## Рекомендуемые обозначения

Для сущностей использовать прямоугольники с понятными названиями на русском языке. Для связей использовать линии с указанием кратности:

- 1:1 - один к одному;
- 1:M - один ко многим;
- M:N - многие ко многим.

Связь между пользователями и играми рекомендуется показывать как связь M:N через сущность "Игровые предпочтения пользователя" или "Игры пользователя".

## Содержание диаграммы

Диаграмма не должна перегружаться физическими деталями реализации. Не требуется указывать все технические поля таблиц, типы данных, индексы и служебные ограничения. Достаточно показать ключевые сущности и смысловые связи между ними.

Допускается указать основные атрибуты сущностей:

- пользователь: идентификатор, VK ID, дата создания;
- профиль пользователя: имя, возраст, город, пол, описание;
- игра: название, код игры;
- фотография: ссылка на фотографию, порядок отображения;
- взаимодействие: отправитель, получатель, действие, дата;
- совпадение: два пользователя, дата совпадения;
- отложенный лайк: отправитель, получатель, сообщение, дата;
- сессия: пользователь, данные сессии, дата обновления.

## Ожидаемый результат

Результатом выполнения задания должна быть концептуальная ER-диаграмма базы данных ВК-бота "ГоПати", отражающая основные сущности предметной области и связи между ними. Диаграмма должна служить переходом от текстового описания структуры базы данных к последующему обоснованию нормализации и физической модели БД.
