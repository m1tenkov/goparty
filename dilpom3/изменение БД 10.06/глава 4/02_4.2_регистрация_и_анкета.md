# 4.2. Реализация логики регистрации и заполнения анкеты

Одним из ключевых сценариев работы ВК-бота ГоПати является регистрация пользователя и последовательное заполнение его анкеты. Данный процесс реализован как пошаговый диалоговый сценарий, в рамках которого бот последовательно запрашивает обязательные данные, контролирует корректность ввода и сохраняет результат в базе данных.

Логика регистрации построена на основе конечного набора состояний диалога. В проекте используются состояния «REG_NAME», «REG_AGE», «REG_GENDER», «REG_CITY», «REG_LOOKING», «STATE_GAMES», «STATE_REG_MICROPHONE», «STATE_ABOUT», «STATE_PHOTO» и промежуточные состояния, связанные с добавлением дополнительных фотографий. Каждое состояние соответствует конкретному этапу заполнения анкеты, а переходы между ними определяются действиями пользователя и результатами валидации введенных данных.

На этапе регистрации пользователь указывает имя, возраст, пол, город, цель поиска, выбирает интересующие игры, указывает, играет ли он с микрофоном, добавляет текстовое описание и фотографии. Для текстовых полей реализована валидация длины и типа содержимого, для возраста проверяется допустимый числовой диапазон, а для фотографий контролируется количество загружаемых изображений. Это позволяет обеспечить единообразие анкет и снизить число ошибок при вводе данных.

Дополнительной особенностью реализации является частичное автозаполнение анкеты из открытых данных профиля ВКонтакте. Если пользователь еще не ввел имя, город, пол или возраст, а эти сведения доступны через VK API, бот может использовать их как значения по умолчанию. Такой подход сокращает количество ручных действий и делает первичное заполнение анкеты более удобным.

Выбор игр реализован через inline-кнопки, а не через свободный текстовый ввод. Пользователь отмечает интересующие его позиции из заранее определенного справочника игр, после чего выбор сохраняется в связующей таблице «user_games». Факт заполнения игрового блока определяется наличием хотя бы одной выбранной игры, поэтому отдельный технический флаг для этого этапа не требуется.

Новая анкета по умолчанию создается неактивной: поле «profiles.is_active» имеет значение «0». Пока пользователь не заполнил все обязательные сведения, такая анкета хранится в базе как черновик и не попадает в выдачу другим пользователям. После прохождения всех обязательных шагов бот публикует анкету, устанавливая «is_active = 1». Такой подход исключает ситуацию, при которой незавершенная анкета может быть показана в подборе.

Фотографии сохраняются в таблице «user_photos». Для каждой фотографии хранится локальный путь «photo_path», порядок отображения «sort_order» и технический токен «vk_photo_token». При отправке анкеты бот сначала пытается использовать сохраненный VK-токен. Если он недоступен, фотография загружается с сервера по локальному пути, после чего новый токен сохраняется в базе для последующих отправок.

После завершения регистрации бот показывает пользователю экран собственной анкеты. На этом этапе пользователь может подтвердить корректность данных, перейти к просмотру анкет, открыть редактирование профиля или временно отключить анкету от участия в подборе. Таким образом, регистрационный сценарий не только собирает данные, но и переводит анкету из состояния черновика в состояние опубликованного профиля.
