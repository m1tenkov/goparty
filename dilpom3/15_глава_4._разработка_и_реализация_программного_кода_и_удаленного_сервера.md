## 4.1. Реализация серверной части ВК-бота

Серверная часть проекта ГоПати реализована на языке Python и представляет собой приложение, обеспечивающее прием событий от платформы «ВКонтакте», маршрутизацию пользовательских действий, взаимодействие с базой данных и отправку ответных сообщений [14]. На уровне архитектуры решение разделено на несколько логических компонентов: HTTP-точка входа для Callback API, обработчик событий, модуль работы с VK API, слой бизнес-логики диалога и слой доступа к данным [5; 7; 17; 18].

В качестве внешней точки входа используется приложение «FastAPI» [19], реализованное в файле «app.py». Данный компонент принимает HTTP-запросы от платформы «ВКонтакте» по маршруту «/vk/callback», проверяет корректность секретного ключа «VK_CALLBACK_SECRET», обрабатывает событие подтверждения сервера через «confirmation token», а для рабочих событий помещает полезную нагрузку во внутреннюю очередь обработки. Дополнительно в приложении реализован технический endpoint «/health», позволяющий выполнять проверку доступности сервиса после развертывания.

Такая реализация позволяет быстро подтвердить получение события со стороны VK и не выполнять длительную пользовательскую логику непосредственно внутри HTTP-запроса. Если событие относится к типам «message_new» или «message_event», оно помещается во внутреннюю очередь. Если очередь заполнена, приложение возвращает ошибку 503, что позволяет явно зафиксировать ситуацию перегрузки.

Обработка входящих событий вынесена в модуль «event_processing.py». В рамках данного слоя различаются два основных типа событий: «message_new», соответствующий обычным входящим сообщениям пользователя, и «message_event», соответствующий callback-событиям от inline-кнопок. Также модуль содержит внутреннюю очередь callback-событий и worker-поток, который запускается при старте приложения, забирает события из очереди и передает их в прикладные обработчики. Такой подход позволяет FastAPI быстро вернуть платформе «ВКонтакте» ответ «ok», а пользовательский сценарий обработать отдельно, не задерживая callback-запрос.

Для взаимодействия с API социальной сети используется модуль «vk_bot.py», создающий объект VK API на основе токена сообщества [16; 17]. Через данный объект выполняется отправка сообщений пользователям, ответ на callback-события, получение открытых данных профиля и другие сетевые вызовы, необходимые для функционирования бота.

Центральный прикладной компонент серверной части размещен в модуле «bot_handlers/router.py». Именно он реализует основную диалоговую логику: стартовое взаимодействие, последовательное заполнение анкеты, показ собственной анкеты, просмотр анкет других пользователей, работу с лайками, жалобами, мэтчами, редактированием профиля и восстановлением состояния после повторного обращения пользователя.

Таким образом, серверная часть ГоПати построена по модульному принципу и сочетает в себе HTTP-интерфейс на базе «FastAPI», интеграцию с «VK API», очередь обработки callback-событий, прикладную обработку пользовательских сценариев и отдельный слой работы с данными. Такая архитектура обеспечивает разделение ответственности между компонентами, упрощает сопровождение проекта и создает основу для дальнейшего расширения функциональности.
