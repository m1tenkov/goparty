## 3.2. Обоснование выбора типа СУБД

Выбор реляционной СУБД MySQL для реализации системы ГоПати обусловлен особенностями предметной области и структурой хранимых данных. Разрабатываемый ВК-бот оперирует несколькими взаимосвязанными сущностями: пользователи, анкеты, игровые предпочтения, фотографии, лайки, мэтчи и пользовательские сессии. Между этими сущностями существуют устойчивые логические связи: пользователь связан с анкетой, анкета связана с набором выбранных игр и фотографий, а взаимодействия пользователей формируют историю действий, взаимные совпадения и отложенные лайки. Жалобы пользователей отправляются в чат модерации, при этом в базе данных фиксируется отрицательное взаимодействие с анкетой. В таких условиях реляционная модель данных является наиболее подходящей, поскольку она обеспечивает удобное представление структурированной информации, поддержку связей между таблицами и контроль целостности данных [2; 3; 12].

Дополнительным фактором выбора MySQL стала необходимость надежного хранения данных при многократных обращениях к системе, а также возможность выполнения выборок по различным критериям. Для бота важно быстро получать анкету пользователя, находить подходящих кандидатов по заданным параметрам, сохранять действия пользователей и восстанавливать состояние после перезапуска системы. Реляционная СУБД позволяет эффективно реализовать подобные сценарии за счет использования первичных и внешних ключей, индексов, ограничений целостности и стандартного языка запросов SQL [3; 4].

Таким образом, выбор MySQL обусловлен тем, что данная СУБД хорошо подходит для хранения структурированных данных бота, поддерживает сложные взаимосвязи между сущностями и обеспечивает необходимую надежность при реализации серверной логики приложения.
